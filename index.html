Introduction

Kubernetes (K8s) is an open-source container orchestration platform originally developed by Google and now maintained by the CNCF (Cloud Native Computing Foundation). It automates deployment, scaling, and lifecycle management of containerized applications.

Why Kubernetes? Docker gives you containers. Kubernetes gives you a system to run those containers reliably across many machines — with automatic restarts, load balancing, rolling updates, and self-healing built in.

You write a YAML manifestDescribes the desired state — what containers to run, how many replicas, what ports to expose
kubectl apply -f your-app.yamlYou submit it to the API Server on the Master node
Scheduler places pods on Worker nodesBased on resource availability and constraints
Kubernetes keeps it running foreverRestarts crashed containers, reschedules if a node fails, scales up/down on demand

Kubernetes Components — what each piece does

🖥️ Control Plane Components (Master Node)

🌐
kube-apiserver
API Server
The front door of Kubernetes. Every kubectl command, every dashboard click, every internal component — all talk through the API Server. It validates requests and updates the cluster state in etcd.
REST API Authentication Authorization Admission Control
🗄️
etcd
Distributed Key-Value Store
The brain/memory of the cluster. All cluster state, configuration, secrets, and service discovery data is stored here. It's a distributed key-value store — if you lose etcd, you lose everything.
Cluster State Configs Secrets Leader Election
📅
kube-scheduler
Scheduler
The pod placement engine. When a new pod needs to run, the Scheduler picks the best Worker node based on CPU/memory availability, affinity rules, taints/tolerations, and other constraints.
Node Selection Resource Fitting Affinity Rules Taints/Tolerations
🔁
kube-controller-manager
Controller Manager
Runs background control loops that watch cluster state and make changes to reach the desired state. Includes the Node Controller, Replication Controller, Endpoints Controller, and many others.
Node Controller ReplicaSet Endpoints ServiceAccount

⚙️ Worker Node Components

🤖
kubelet
Node Agent
The agent running on every worker node. It receives pod specs from the API Server and ensures the containers described in those specs are running and healthy. Reports node and pod status back to master.
Pod Lifecycle Health Checks Resource Reporting Volume Mounting
🔀
kube-proxy
Network Proxy
Maintains network rules on every node. Handles routing of traffic to the right pod when you access a Kubernetes Service. Uses iptables or IPVS to implement virtual IPs for services.
Service Routing iptables Load Balancing Network Rules
🐳
Container Runtime
Docker / containerd / CRI-O
The software that actually runs containers. Kubernetes itself doesn't run containers — it tells the container runtime to do it. Docker CE is used in this guide, but containerd and CRI-O are also supported.
Pull Images Start/Stop Containers CRI Interface Namespaces
🌩️
CNI Plugin (Weave Net)
Network Plugin
Provides pod-to-pod networking across nodes. Without a CNI plugin, pods on different nodes can't communicate. Weave Net creates an overlay network that gives every pod a unique IP and enables cross-node communication.
Pod Networking Overlay Network DNS Resolution Network Policies

📦 Kubernetes Objects (Workload Resources)

ObjectWhat it doesWhen to use
PodSmallest deployable unit. One or more containers that share network/storageRarely directly — use Deployments instead
DeploymentManages a set of identical pods. Handles rolling updates and rollbacksStateless apps (web servers, APIs)
ReplicaSetEnsures a specified number of pod replicas are running at all timesUsed internally by Deployments
ServiceStable network endpoint for pods. LoadBalancer, NodePort, or ClusterIPAny time you need to expose a pod
ConfigMapStore non-sensitive config data (env vars, config files)App configuration separate from code
SecretStore sensitive data (passwords, tokens, keys) — base64 encodedDB passwords, API keys, TLS certs
NamespaceVirtual clusters within a cluster. Isolates resourcesMulti-team or multi-environment setups
PersistentVolumeStorage resource in the cluster (NFS, cloud disk, etc.)Databases and stateful apps
IngressManages external HTTP/HTTPS access to servicesRouting web traffic to multiple services
DaemonSetEnsures a pod runs on every (or selected) nodeLog collectors, monitoring agents
StatefulSetLike Deployment but with stable identity and storage per podDatabases (MySQL, MongoDB, Cassandra)
Job / CronJobRuns pods to completion. CronJob runs on a scheduleBatch processing, backups, migrations

Pre-Installation Steps
ℹ️

Run all commands in this section on BOTH Master and all Worker nodes.

5.1 — Update System Packages

bash — all nodes
sudo apt-get update

5.2 — Disable Swap

Kubernetes requires swap to be disabled — kubelet will refuse to start if swap is active.

bash — all nodes
sudo swapoff -a

Make permanent by commenting out the swap entry in /etc/fstab:

bash — all nodes
sudo nano /etc/fstab
# Comment out any line with 'swap', e.g.:
# /swapfile   none   swap   sw   0   0

5.3 — Set Unique Hostname per Node

bash — all nodes (different name each)
sudo nano /etc/hostname
# Master: k8s-master
# Workers: k8s-worker-01, k8s-worker-02 ...

5.4 — Note Private IP of Each Node

bash — all nodes
ip addr

Master Node Setup
Step 1Install Docker CE on Master
bash — master only
sudo apt-get update

sudo apt-get install -y \
    apt-transport-https \
    ca-certificates \
    curl \
    gnupg-agent \
    software-properties-common

curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -

sudo add-apt-repository \
    "deb [arch=amd64] https://download.docker.com/linux/ubuntu \
    $(lsb_release -cs) \
    stable"

sudo apt-get update
sudo apt-get install docker-ce docker-ce-cli containerd.io -y
Step 2Install kubeadm, kubectl, kubelet on Master
📌

Use version 1.21.3-00. Version 1.24.1-00 has known installation issues — avoid it.

bash — master only
apt-get update && apt-get install -y apt-transport-https curl

curl -s https://packages.cloud.google.com/apt/doc/apt-key.gpg | apt-key add -

cat < /etc/apt/sources.list.d/kubernetes.list
deb https://apt.kubernetes.io/ kubernetes-xenial main
EOF

apt-get update

sudo apt-get install -y \
    kubectl=1.21.3-00 \
    kubeadm=1.21.3-00 \
    kubelet=1.21.3-00

# Lock versions to prevent accidental upgrades
apt-mark hold kubelet kubeadm kubectl

What are these packages?

PackageRole
kubeadmBootstrap tool — initializes the cluster master and joins workers
kubectlCLI tool to interact with the cluster (apply, get, delete, logs...)
kubeletAgent that runs on every node — manages pods and containers
Step 3Initialize the Master Node (kubeadm init)
bash — master only
sudo kubeadm init
📌

IMPORTANT: Copy and save the kubeadm join ... command printed at the end — you need it to add Worker nodes.

Troubleshooting — cgroupdriver Error

bash — cgroupdriver fix
sudo nano /etc/docker/daemon.json

# Paste this content, then Ctrl+X > Y > Enter:
{
  "exec-opts": ["native.cgroupdriver=systemd"]
}

sudo systemctl restart docker
sudo kubeadm reset
sudo kubeadm init
Step 4Configure kubectl Access
bash — master only
mkdir -p $HOME/.kube
sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
sudo chown $(id -u):$(id -g) $HOME/.kube/config

Step 2Install Kubernetes Packages on Worker
bash — each worker
apt-get update && apt-get install -y apt-transport-https curl

curl -s https://packages.cloud.google.com/apt/doc/apt-key.gpg | apt-key add -

cat < /etc/apt/sources.list.d/kubernetes.list

apt-mark hold kubelet kubeadm kubectl
Step 3Join Worker to the Cluster
bash — each worker (use your actual values)
sudo kubeadm join <MASTER_IP>:6443 \
    --token <TOKEN> \
    --discovery-token-ca-cert-hash sha256:<HASH>
📌

Replace <MASTER_IP>, <TOKEN>, <HASH> with actual values from master's kubeadm init output. To regenerate: run kubeadm token create --print-join-command on master.

Verify the Cluster
bash — master
# List all nodes
kubectl get nodes -o wide

# Watch continuously (Ctrl+C to stop)
kubectl get nodes -o wide -w

# Check system pods are running
kubectl get pods -n kube-system
✅

All nodes should show STATUS: Ready after CNI is installed (Section 9).

9
Install CNI Plugin — Weave Net

Without a CNI plugin, pods on different nodes cannot communicate and nodes stay in NotReady state. Weave Net creates an overlay network that gives every pod a unique IP across all nodes.

bash — master only
kubectl apply -f https://github.com/weaveworks/weave/releases/download/v2.8.1/weave-daemonset-k8s.yaml
📌

Wait ~60 seconds, then run kubectl get nodes — all nodes should show STATUS: Ready.

Multi-Network Configuration (Optional)

If a worker has multiple network interfaces and needs a specific IP for cluster traffic:

bash — worker node
sudo nano /etc/systemd/system/kubelet.service.d/10-kubeadm.conf

# Add inside [Service] section:
Environment="KUBELET_KUBECONFIG_ARGS=--bootstrap-kubeconfig=/etc/kubernetes/bootstrap-kubelet.conf \
  --kubeconfig=/etc/kubernetes/kubelet.conf --node-ip=<WORKER_PRIVATE_IP>"

sudo systemctl daemon-reload
sudo systemctl restart kubelet

Kubernetes Dashboard

A web UI to visually manage your cluster — view pod logs, deploy apps, scale workloads, and monitor resources.

Step 1Deploy the Dashboard
bash — master
kubectl apply -f https://raw.githubusercontent.com/kubernetes/dashboard/v2.0.0/aio/deploy/recommended.yaml
Step 2Create Admin Service Account
bash — master
kubectl create serviceaccount dashboard-admin-sa

kubectl create clusterrolebinding dashboard-admin-sa \
    --clusterrole=cluster-admin \
    --serviceaccount=default:dashboard-admin-sa
Step 3Get Login Token
bash — master
kubectl get secret \
  $(kubectl get serviceaccount dashboard-admin-sa -o jsonpath="{.secrets[0].name}") \
  -o jsonpath="{.data.token}" | base64 --decode
Step 4Access Dashboard in Browser
bash — master
kubectl proxy

Open this URL:

browser URL
http://localhost:8001/api/v1/namespaces/kubernetes-dashboard/services/https:kubernetes-dashboard:/proxy/

Deploy Applications

Once the cluster is running, here's how to deploy and manage your applications.

Deploy an Nginx Web Server (Example)

bash — create deployment
# Create a deployment with 3 replicas
kubectl create deployment nginx --image=nginx --replicas=3

# Expose it as a NodePort service
kubectl expose deployment nginx --port=80 --type=NodePort

# Check status
kubectl get pods
kubectl get service nginx

Deploy via YAML Manifest (Recommended)

yaml — save as app.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
  labels:
    app: my-app
spec:
  replicas: 2
  selector:
    matchLabels:
      app: my-app
  template:
    metadata:
      labels:
        app: my-app
    spec:
      containers:
      - name: my-app
        image: nginx:latest
        ports:
        - containerPort: 80
        resources:
          requests:
            memory: "64Mi"
            cpu: "250m"
          limits:
            memory: "128Mi"
            cpu: "500m"
---
apiVersion: v1
kind: Service
metadata:
  name: my-app-service
spec:
  selector:
    app: my-app
  type: NodePort
  ports:
  - port: 80
    targetPort: 80
    nodePort: 30080
bash — apply the manifest
# Deploy
kubectl apply -f app.yaml

# Check deployment status
kubectl get deployments
kubectl get pods -o wide

# Scale up to 5 replicas
kubectl scale deployment my-app --replicas=5

# Rolling update to new image version
kubectl set image deployment/my-app my-app=nginx:1.21

# Rollback if something goes wrong
kubectl rollout undo deployment/my-app

# Delete deployment
kubectl delete -f app.yaml

Namespace Management

bash — namespaces
# Create a namespace
kubectl create namespace production

# Deploy into a specific namespace
kubectl apply -f app.yaml -n production

# List pods in a namespace
kubectl get pods -n production

# List all pods in all namespaces
kubectl get pods --all-namespaces

Networking — Services & Ingress

Service Types Explained

TypeDescriptionUse Case
ClusterIPInternal cluster IP only — not reachable outsideInternal microservice-to-microservice traffic
NodePortExposes on a port on every node (30000–32767)Dev/testing — direct node access
LoadBalancerProvisions cloud load balancer (AWS, GCP, Azure)Production on cloud providers
ExternalNameMaps service to external DNS nameRouting to external services

Useful Networking Commands

bash — networking
# List all services
kubectl get services --all-namespaces

# Describe a service (see endpoints, ports)
kubectl describe service <service-name>

# Get pod IP addresses
kubectl get pods -o wide

# Port-forward a pod locally for testing
kubectl port-forward pod/<pod-name> 8080:80

# View logs of a pod
kubectl logs <pod-name>

# Follow logs in real-time
kubectl logs -f <pod-name>

# Exec into a running pod
kubectl exec -it <pod-name> -- /bin/bash

Troubleshooting

Check Component Health

bash — diagnostics
# Check cluster component status
kubectl get componentstatuses

# Check node status and conditions
kubectl describe node <node-name>

# Check all system pods
kubectl get pods -n kube-system

# Check kubelet service status
sudo systemctl status kubelet

# View kubelet logs
sudo journalctl -u kubelet -f

# View API server logs
kubectl logs -n kube-system kube-apiserver-<master-hostname>

Pod Troubleshooting

bash — pod issues
# Pod stuck in Pending / CrashLoopBackOff
kubectl describe pod <pod-name>
kubectl logs <pod-name>
kubectl logs <pod-name> --previous   # logs from crashed container

# Get events for troubleshooting
kubectl get events --sort-by=.metadata.creationTimestamp

# Force delete a stuck pod
kubectl delete pod <pod-name> --force --grace-period=0

# Check resource usage on nodes
kubectl top nodes
kubectl top pods

Common Issues & Fixes

ErrorCauseFix
Node NotReadyCNI plugin not installedApply Weave Net (Section 9)
cgroupdriver mismatchDocker and kubelet using different cgroupsSet systemd in daemon.json (Section 6, Step 3)
CrashLoopBackOffContainer keeps crashingRun kubectl logs <pod> --previous to see crash reason
ImagePullBackOffContainer image not found / no accessCheck image name, tag, and registry credentials
Pending podsNo nodes with enough resourcesCheck kubectl describe pod for resource constraints
Connection refused on joinFirewall blocking port 6443Open port 6443 on master: ufw allow 6443

Reset and Reinstall (Nuclear Option)

bash — full reset
# Reset the node (run on any node you want to reset)
sudo kubeadm reset

# Clean up CNI config
sudo rm -rf /etc/cni/net.d
sudo rm -rf $HOME/.kube

# Flush iptables
sudo iptables -F && sudo iptables -t nat -F
sudo iptables -t mangle -F && sudo iptables -X

# Re-initialize master after reset
sudo kubeadm init
